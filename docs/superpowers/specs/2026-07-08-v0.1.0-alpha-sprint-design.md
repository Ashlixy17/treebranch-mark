# v0.1.0-alpha Closing Sprint Design

## Goals

The v0.1.0-alpha closing sprint turns the current MVP pipeline into the first releasable alpha.

This sprint should finish only the release-critical work:

1. GitHub Token implementation
2. Source-level memory cache
3. Browser demo verification and screenshot
4. README first-screen rewrite
5. GitHub release notes and alpha tag

Step 8, Timeline Layout, Metro Layout, animation, CLI, GitHub Action, and VS Code extension are outside this sprint.

## Sprint Scope

### 1. GitHub Token

Implement optional GitHub Token support.

Token support is authentication infrastructure and belongs only to the GitHub HTTP client boundary.

The token must not enter:

- `GitSourceInput`
- `GitSourceSnapshot`
- `RenderPipeline.render(input)`
- `RenderPipelineResult`
- Parser
- Graph Builder
- Layout
- RenderModel
- Renderer
- Snapshot JSON output
- `localStorage` except the fixed browser UI key `treebranch.github.token`
- `sessionStorage`
- URL parameters

Invalid tokens should map GitHub `401` responses to `bad-credentials`.

### 2. Source-Level Memory Cache

Implement a first MVP `Snapshot Cache` inside the Source boundary.

Recommended flow:

```text
Browser
  |
  v
RenderPipeline
  |
  v
GitHubApiSource
  |
  v
SourceCache
  |
  v
GitHubRestClient
  |
  v
GitHub API
```

Pipeline must not know that cache exists.

Parser, Graph Builder, Layout, RenderModel, and Renderer must not know that cache exists.

The MVP cache stores `GitSourceSnapshot` values in memory.

Suggested interface:

```ts
interface SourceCache<TValue> {
  get(key: string): TValue | undefined
  set(key: string, value: TValue): void
  clear(): void
}
```

First implementation:

```ts
class MemorySourceCache<TValue> implements SourceCache<TValue>
```

MVP TTL:

```text
5 minutes
```

Cache key should include normalized source parameters:

```text
github:
owner/repo
branch
maxCommitsPerBranch
includePullRequests
includeContributors
includeTags
```

Token must not be part of the cache key. Token authenticates the request; it does not change the public repository snapshot.

HTTP caching with ETag, `If-None-Match`, and `304 Not Modified` is intentionally deferred.

## Source Infrastructure Principle

Source is responsible for data acquisition and all infrastructure directly related to data acquisition:

- Authentication
- Cache
- Retry
- Rate limiting
- Request optimization
- Error mapping

Pipeline only consumes `GitSourceSnapshot`.

Pipeline does not care whether data came from a network request, cache, authenticated request, local Git command, GitLab, or a future provider.

## Testing Requirements

Automated tests must cover:

- Token header is omitted when token is missing.
- Token header is included when token is provided.
- Empty or whitespace token is ignored.
- GitHub `401` maps to `bad-credentials`.
- Token is not serialized into snapshots or pipeline results.
- Token is not accepted by `GitSourceInput`.
- Token is not accepted by `RenderPipeline.render(input)`.
- Cache hit avoids duplicate GitHub API work.
- Cache miss loads from GitHub API and stores the snapshot.
- Cache respects `cache: 'reload'`.
- Cache respects `cache: 'no-store'`.
- Cache key does not contain token.

Manual release verification must include:

```bash
npm test
npm run build
npm run lint
```

## Definition of Done

v0.1.0-alpha closing sprint is complete when:

- Optional GitHub Token works in the browser.
- Invalid token errors are localized as `bad-credentials`.
- Token is persisted only in `localStorage` key `treebranch.github.token`, and is never serialized, logged, or included in public input/result types.
- Source-level memory cache works and is covered by tests.
- Browser can generate and display an SVG for at least one public repository.
- README first screen is release-ready.
- Demo screenshot exists at `docs/assets/v0.1.0-alpha-demo.png`.
- Release notes exist at `docs/releases/v0.1.0-alpha.md`.
- ADR files exist for the core architecture decisions.
- `npm test`, `npm run build`, and `npm run lint` pass.
- `main` contains release-ready code.
- `v0.1.0-alpha` tag is created after final verification.
