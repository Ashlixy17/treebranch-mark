# Treebranch Mark v0.1.0-alpha

`v0.1.0-alpha` is the first public alpha milestone for Treebranch Mark. It proves the full browser-based rendering pipeline from GitHub repository data to a generated SVG branch graph.

## Highlights

- Complete end-to-end rendering pipeline
- GitHub Source Layer
- Commit DAG Parser
- Branch Graph Builder
- Tree Layout
- RenderModel
- SVG Renderer
- Browser Viewer
- Optional GitHub Personal Access Token support
- Source-level Memory Snapshot Cache
- Circular GitHub avatar commit nodes with circle fallback

## Architecture

```text
Source
  -> Parser
  -> Graph Builder
  -> Tree Layout
  -> RenderModel
  -> SVG Renderer
  -> Browser Viewer
```

The browser UI calls a shared `RenderPipeline`, so future frontends such as CLI, GitHub Action, and VS Code Extension can reuse the same core flow.

## What's Included

- Load public GitHub repositories through the GitHub REST API
- Normalize GitHub data into a serializable `GitSourceSnapshot`
- Parse commits into a `CommitGraph`
- Build a `BranchGraph` from branch heads and reachable commits
- Generate a deterministic MVP Tree Layout
- Convert layout data into a renderer-agnostic RenderModel
- Render the graph as a standalone SVG string
- Display the generated SVG in the browser
- Support optional GitHub Personal Access Token authentication to reduce rate-limit issues
- Reuse GitHub snapshots from a five-minute in-memory cache
- Render commit authors as circular GitHub avatars when available

## Known Limitations

- Tree Layout only; Timeline and Metro layouts are not implemented yet
- Local Git Source is planned for v0.2
- GitLab Source is planned for future versions
- The SVG renderer remains intentionally minimal: lines, avatar images, fallback circles, and text
- No zoom, pan, hover tooltip, PNG export, GIF export, or animation yet

## Next

- CLI
- Local Git Source

# Treebranch Mark v0.1.0-alpha 简体中文说明

`v0.1.0-alpha` 是 Treebranch Mark 的第一个公开 Alpha 里程碑。这个版本验证了从 GitHub 仓库数据到 SVG 分支图生成的完整浏览器端渲染流程。

## 亮点

- 完整的端到端渲染 Pipeline
- GitHub Source Layer
- Commit DAG Parser
- Branch Graph Builder
- Tree Layout
- RenderModel
- SVG Renderer
- Browser Viewer
- 支持可选的 GitHub Personal Access Token
- Source 层 Memory Snapshot Cache
- 圆形 GitHub 贡献者头像 Commit 节点，并支持圆点回退

## 架构

```text
Source
  -> Parser
  -> Graph Builder
  -> Tree Layout
  -> RenderModel
  -> SVG Renderer
  -> Browser Viewer
```

浏览器界面通过统一的 `RenderPipeline` 调用核心流程。后续 CLI、GitHub Action、VS Code Extension 等入口都可以复用同一套核心渲染链路。

## 本版本包含

- 通过 GitHub REST API 加载公开 GitHub 仓库
- 将 GitHub 数据标准化为可序列化的 `GitSourceSnapshot`
- 将提交解析为 `CommitGraph`
- 根据分支 HEAD 和可达提交构建 `BranchGraph`
- 生成确定性的 MVP Tree Layout
- 将布局数据转换为与渲染器无关的 RenderModel
- 将图渲染为独立的 SVG 字符串
- 在浏览器中展示生成的 SVG
- 支持可选 GitHub Personal Access Token，用于缓解 GitHub API 限流问题
- 使用五分钟内存缓存复用 GitHub Snapshot
- Commit 存在 GitHub 用户信息时显示圆形贡献者头像

## 已知限制

- 当前仅支持 Tree Layout，Timeline 和 Metro Layout 尚未实现
- Local Git Source 计划在 v0.2 中实现
- GitLab Source 计划在未来版本中实现
- SVG Renderer 仍保持最小实现：仅包含线条、头像、回退圆点和文字
- 暂不支持缩放、拖拽、悬停提示、PNG 导出、GIF 导出或动画

## 下一步

- CLI
- Local Git Source
