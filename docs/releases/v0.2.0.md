# Treebranch Mark v0.2.0

`v0.2.0` 将 Treebranch Mark 从仅支持 GitHub API 的浏览器工具扩展为同时支持本地 Git 仓库的可视化引擎。

## Highlights

- 新增 Node-only `LocalGitSource`，直接读取本地 Git 仓库。
- 新增 `treebranch render <path>` CLI，可将本地仓库输出为 SVG。
- GitHub API 与 Local Git 共用统一的 `GitSourceSnapshot` 数据契约。
- RenderPipeline 支持根据 Source 自动推导输入类型。
- 支持本地分支、Merge Commit、detached HEAD、裸仓库和多行 Commit 信息。
- 本地 Git 不包含头像时自动回退为圆形 Commit 节点。
- Browser 与 CLI 使用独立构建产物，并具有自动化边界验证。

## Known Limitations

- Local Git MVP 只读取 `refs/heads/*`。
- 暂不包含 remote-tracking branches、tags 和 submodules。
- CLI 尚未发布为独立 npm package，需要从源码构建或使用 `npm link`。
- 当前仍只支持 Tree Layout。

# Treebranch Mark v0.2.0

`v0.2.0` expands Treebranch Mark from a GitHub API browser tool into a visualization engine that can also render local Git repositories.

## Highlights

- Added a Node-only `LocalGitSource` for reading local Git repositories.
- Added the `treebranch render <path>` CLI for generating SVG from local repositories.
- GitHub API and Local Git now share the same `GitSourceSnapshot` contract.
- RenderPipeline infers its input type from the configured Source.
- Added support for local branches, merge commits, detached HEAD, bare repositories, and multiline commit messages.
- Local commits without avatars automatically fall back to circle nodes.
- Browser and CLI builds are isolated and protected by automated boundary checks.

## Known Limitations

- The Local Git MVP reads only `refs/heads/*`.
- Remote-tracking branches, tags, and submodules are not included yet.
- The CLI is not yet distributed as a standalone npm package; build from source or use `npm link`.
- Tree Layout remains the only layout.
