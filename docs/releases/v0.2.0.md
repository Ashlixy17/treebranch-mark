# Treebranch Mark v0.2.0

`v0.2.0` 为 Treebranch Mark 增加了本地 Git 仓库支持。现在除了在浏览器中读取 GitHub 仓库，还可以通过 Node CLI 直接读取本地 `.git` 并生成 SVG，不再受 GitHub API 限流影响。

## Highlights

- 新增 Node-only `LocalGitSource`，通过系统 Git 读取本地仓库。
- 新增 `treebranch render <path>` CLI，可直接生成 SVG 文件。
- GitHub API 与 Local Git 共用统一的 `GitSourceSnapshot` 数据契约和 RenderPipeline。
- 支持本地分支、Merge Commit、detached HEAD、裸仓库及多行 Commit 信息。
- 本地 Commit 没有头像时自动回退为圆形节点。
- Browser 与 CLI 使用独立构建产物，并加入自动化边界验证。

## 快速使用

```bash
npm install
npm run build
node dist-cli/treebranch.js render . --output branch.svg
```

需要 Node.js `>=20.19.0`，并确保 Git 已加入 `PATH`。

## Known Limitations

- Local Git MVP 只读取 `refs/heads/*`。
- 暂不包含 remote-tracking branches、tags 和 submodules。
- CLI 尚未发布为独立 npm package，需要从源码构建或使用 `npm link`。
- 当前仍只支持 Tree Layout。

# Treebranch Mark v0.2.0

`v0.2.0` adds local Git repository support to Treebranch Mark. In addition to loading GitHub repositories in the Browser Viewer, the new Node CLI can read a local `.git` repository and generate SVG without relying on GitHub API limits.

## Highlights

- Added a Node-only `LocalGitSource` that reads repositories through the installed Git executable.
- Added the `treebranch render <path>` CLI for generating SVG files.
- GitHub API and Local Git now share the same `GitSourceSnapshot` contract and RenderPipeline.
- Added support for local branches, merge commits, detached HEAD, bare repositories, and multiline commit messages.
- Local commits without avatars automatically fall back to circle nodes.
- Browser and CLI builds are isolated and protected by automated boundary checks.

## Quick Start

```bash
npm install
npm run build
node dist-cli/treebranch.js render . --output branch.svg
```

Requires Node.js `>=20.19.0` and Git available on `PATH`.

## Known Limitations

- The Local Git MVP reads only `refs/heads/*`.
- Remote-tracking branches, tags, and submodules are not included yet.
- The CLI is not yet distributed as a standalone npm package; build from source or use `npm link`.
- Tree Layout remains the only layout.
